# DaddyThermometer

3-канальний монітор температури з LCD-індикацією та сигналізацією. Повна специфікація вимог — у [`vymohy_termometr.md`](vymohy_termometr.md) (FR-1..FR-5, NFR-1); цей README фіксує, що з неї вже реалізовано в коді на поточному етапі.

**Примітка щодо платформи:** специфікація цілить STM32, але поточна прошивка (`platformio.ini`, env `nanoatmega328new`) збирається під Arduino Nano (ATmega328) — це проміжна платформа для розробки/відладки, не фінальне залізо.

## Реалізовано

- **Вимірювання температури (FR-1)** — `ChannelTemp`: 3 незалежні канали, NTC-термістор через дільник напруги, лінеаризація рівнянням Стейнхарта-Гарта. Опитування прив'язане до `Timer(500ms)` у `main.cpp::loop()` (≥2 рази/сек).
- **Виявлення несправності датчика (FR-5)** — `ChannelTemp::status()`: значення нижче `FAULT_TEMP_C` (обрив/коротке) трактується як `Status::Error`, а не як реальна температура.
- **Гістерезис тривоги (FR-4, частково)** — реалізовано в `ChannelTemp::status()`: вихід зі стану `TooHigh`/`TooLow` лише після повернення в межу із запасом `HYSTERESIS_C` (0.5°C). Звукова сигналізація (PWM-тон, кнопка acknowledge) ще **не** реалізована.
- **Відображення (FR-2)** — `MonitorRow`: один рядок LCD на канал, власні CGRAM-символи `↑`/`↓` для напрямку виходу за межі, `ERR` для несправності. Рендер диференційний (пише в LCD лише те, що змінилось), рядок у тривозі блимає цілком за власним `Timer`, незалежним для кожного рядка.
- **Меню налаштування меж (FR-3, частково)** — стек сторінок без динамічного виділення пам'яті (`MenuController` + `MenuPage`):
  - `ChannelListPage` — вибір каналу (K1/K2/K3), керований масивом `Entry`, а не хардкодом;
  - `LimitEditPage` — послідовне редагування нижньої, потім верхньої межі одного каналу, з валідацією `low < high`;
  - навігація трьома фізичними кнопками (Up/Down/Ok): довге утримання OK поза меню відкриває його (вхідна сторінка — список каналів), одиночний клік OK усередині меню підтверджує поле, подвійний клік — "назад".
  - Межі поки зберігаються лише в оперативній пам'яті об'єкта `ChannelTemp` — енергонезалежного збереження (EEPROM) ще немає.

## Ще не реалізовано

- Звукова сигналізація та кнопка вимкнення звуку (FR-4)
- Збереження меж у non-volatile пам'яті + відновлення після перезавантаження (FR-3, NFR-1)
- Вільний вибір, яку саме межу редагувати, замість жорсткої послідовності "спочатку нижня, потім верхня" (обговорюється, у розробці)
- Юніт-тести (`test/` — порожній скелет)

## Архітектура

| Файл | Відповідальність |
|---|---|
| `src/ChannelTemp.*` | Вимірювання одного каналу + обчислення стану (Ok/TooLow/TooHigh/Error) |
| `src/MonitorRow.*` | Рендер одного рядка LCD у режимі моніторингу (значення, статус, блимання) |
| `src/Timer.*` | Неблокуючий таймер (інтервал + `ready()`) |
| `src/Status.h` | Спільний `enum class Status`, використовується `ChannelTemp` і `MonitorRow` |
| `src/menu/MenuController.*` | Стек сторінок меню, розбір кнопок Up/Down/Ok/Back |
| `src/menu/MenuPage.h` | Базовий клас сторінки меню (`onEnter`/`render`/`onButton`) |
| `src/menu/ChannelListPage.*` | Сторінка вибору каналу |
| `src/menu/LimitEditPage.*` | Сторінка редагування меж каналу |
| `src/main.cpp` | Точка входу: `setup()`/`loop()`, зв'язує канали, LCD, кнопки й меню разом |

## Збірка

PlatformIO, середовище `nanoatmega328new`:

```bash
pio run              # збірка
pio run -t upload    # прошивка
pio device monitor    # серійний монітор (9600 бод)
```

Залежності (`platformio.ini`): `marcoschwartz/LiquidCrystal_I2C`, `gyverlibs/EncButton`.
