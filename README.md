# DaddyThermometer

3-канальний монітор температури з LCD-індикацією та звуковою сигналізацією. Повна специфікація вимог — у [`vymohy_termometr.md`](vymohy_termometr.md) (FR-1..FR-5, NFR-1..NFR-5); цей README фіксує, що з неї вже реалізовано в коді на поточному етапі.

**Примітка щодо платформи:** специфікація цілить STM32, але поточна прошивка (`platformio.ini`, env `nanoatmega328new`) збирається під Arduino Nano (ATmega328) — це проміжна платформа для розробки/відладки, не фінальне залізо.

## Реалізовано

- **Вимірювання температури (FR-1)** — `ChannelTemp`: 3 незалежні канали, NTC-термістор через дільник напруги, лінеаризація рівнянням Стейнхарта-Гарта. Опитування прив'язане до `Timer(500ms)` (≥2 рази/сек).
- **Відображення (FR-2)** — `MonitorRow`: один рядок LCD на канал, власні CGRAM-символи `↑`/`↓` для напрямку виходу за межі, `ERR` для несправності. Рендер диференційний (пише в LCD лише те, що змінилось), рядок у тривозі блимає цілком за власним `Timer`, незалежним для кожного рядка.
- **Меню налаштування меж (FR-3)** — стек сторінок без динамічного виділення пам'яті (`MenuController` + `MenuPage`, `src/menu/`):
  - `ChannelListPage` — вибір каналу (K1/K2/K3), керований масивом `Entry`, а не хардкодом;
  - `Limit.h/.cpp` (`LimitSelectPage` + `LimitValueEditPage`) — вибір поля (нижня межа / верхня межа / Save) і редагування обраного; валідація `low < high`; кожне поле показує `*`, якщо не збережене, а рядок Save — `"Nothing to save"` й недоступний для вибору, поки нема незбережених змін;
  - навігація трьома фізичними кнопками (Up/Down/Ok): довге утримання OK поза меню відкриває меню (вхідна сторінка — список каналів), одиночний клік OK усередині меню підтверджує/заходить, подвійний клік — "назад"; утримання Up/Down повторює дію (EncButton `step()`) замість повторних окремих кліків;
  - **автовихід (розділ 4-А)** — 20с бездіяльності в меню повертають у режим моніторингу; незбережене редагування відкидається саме тим, що жодна сторінка не встигла отримати свій OK/Save.
- **Збереження меж (FR-3, NFR-4)** — `EepromLimitStore`: запис лише в момент явного підтвердження Save, з magic-байтом проти сміття на чистому EEPROM; читання при старті (`setup()`) відновлює межі після перезавантаження.
- **Звукова сигналізація (FR-4)** — `Buzzer`: не рівний тон, а "виття" — перемикання 3000/3500 Гц кожні 150мс через `tone()`/`noTone()`, немає жодних затримок (`loop()` не блокується). Гістерезис тривоги — в `ChannelTemp::status()` (вихід зі стану `TooHigh`/`TooLow` лише після повернення в межу із запасом `HYSTERESIS_C`, 0.5°C). Окрема кнопка Mute глушить звук поточної тривоги (`ChannelTemp::acknowledge_sound()`), лишаючи візуальну індикацію; тривога на новому каналі (або зміна типу тривоги, напр. TooHigh → Error) знову вмикає звук — прапорець ack зберігається на кожен канал окремо, а не один спільний на пристрій. Сигналізація й гістерезис оновлюються щоцикл незалежно від того, чи користувач у меню налаштувань.
- **Виявлення несправності датчика (FR-5)** — `ChannelTemp::status()`: значення нижче `FAULT_TEMP_C` (обрив/коротке) трактується як `Status::Error`, а не як реальна температура; трактується як аварійний стан (сигналізація + індикація), як і вихід за межі.

## Ще не реалізовано

- **NFR-2 (надійність)** — watchdog-таймер відсутній; відновлення після помилок шини/датчика без повного перезавантаження — лише частково (`Status::Error` сам знімається, коли термістор повертає фізичне значення).
- **NFR-5 (опційно)** — діагностичний вивід по UART: `Serial.begin()` викликається, але нічого не друкується.
- Косметика: рядок 4 LCD — статичний напис замість динамічного статусу/підказок кнопок; одиниця виміру друкується як `"C"`, не `"°C"`; `LimitSelectPage` без заголовка-якоря.
- Платформа в коді (AVR) досі розходиться з платформою в специфікації (STM32) — рішення не зафіксоване в самому документі вимог.
- Юніт-тести (`test/` — порожній скелет).

## Архітектура

| Файл | Відповідальність |
|---|---|
| `src/ChannelTemp.*` | Вимірювання одного каналу, межі, гістерезисний стан (Ok/TooLow/TooHigh/Error), dirty/ack-прапорці для збереження й звуку |
| `src/EepromLimitStore.*` | Читання/запис меж каналу в EEPROM (NFR-4) |
| `src/Buzzer.*` | Неблокуючий "виючий" сигнал на пасивному п'єзо (FR-4) |
| `src/MonitorRow.*` | Рендер одного рядка LCD у режимі моніторингу (значення, статус, блимання) |
| `src/Timer.*` | Неблокуючий таймер (інтервал + `ready()`) |
| `src/Status.h` | Спільний `enum class Status`, використовується `ChannelTemp` і `MonitorRow` |
| `src/menu/MenuController.*` | Стек сторінок меню, розбір кнопок Up/Down/Ok/Back, автовихід за бездіяльністю |
| `src/menu/MenuPage.h` | Базовий клас сторінки меню (`onEnter`/`render`/`onButton`) |
| `src/menu/ChannelListPage.*` | Сторінка вибору каналу |
| `src/menu/Limit.*` | Сторінки вибору поля (`LimitSelectPage`) і редагування значення (`LimitValueEditPage`) |
| `src/main.cpp` | Точка входу: `setup()`/`loop()` (розбитий на `updateSensors`/`updateAlarm`/`updateMenu`/`persistRequestedSaves`/`renderMonitorScreen`), зв'язує канали, LCD, кнопки, бузер і меню разом |

## Збірка

PlatformIO, середовище `nanoatmega328new`:

```bash
pio run              # збірка
pio run -t upload    # прошивка
pio device monitor    # серійний монітор (9600 бод)
```

Залежності (`platformio.ini`): `marcoschwartz/LiquidCrystal_I2C`, `gyverlibs/EncButton`.
